CONTRIBUTING

PAYCHEKX

Thank you for your interest in contributing to PAYCHEKX.

PAYCHEKX is a proprietary product developed and governed by
Artificial Intelligence Capital Ventures (AICV) and founded by Elijah L. Cooley.

This document explains how contributions are evaluated, accepted, and governed.

1. Project Philosophy

PAYCHEKX is built with a strict design philosophy:

Privacy first

No accounts

No tracking

No backend

No analytics

Low liability

High clarity

Minimal complexity

All contributions must preserve these principles.

PAYCHEKX is not an open-source project in the traditional sense.
Contributions are accepted selectively and integrated centrally by AICV.

2. Who May Contribute

Contributions may be accepted from:

Independent developers

Contractors

Designers

Reviewers

Technical advisors

All contributors must agree that:

PAYCHEKX remains proprietary

All accepted contributions become part of AICV intellectual property

No contributor acquires ownership rights in the product

3. Types of Contributions Accepted

We may accept contributions related to:

Bug fixes

UI / UX improvements

Performance optimizations

Accessibility improvements

Documentation improvements

Security reviews

We do not accept:

New product lines

New tools without explicit approval

Changes that introduce accounts, tracking, analytics, or backends

Features that increase legal or regulatory risk

Architectural rewrites without request

4. Design & Architecture Constraints (Strict)

All contributions must preserve:

Static HTML + CSS + Vanilla JavaScript

No external frameworks

No external JS libraries

No build systems

No bundlers

No servers

No APIs

PAYCHEKX must remain:

Deployable on static hosting

Offline-friendly

Deterministic

Easy to audit

5. Code Standards

Contributors must follow:

Plain, readable JavaScript

No obfuscation

No minification in source files

Clear function naming

Inline comments for non-obvious logic

No global namespace pollution

All code must be understandable by a small team without specialized tooling.

6. Submission Process

To propose a contribution:

Fork the repository

Create a feature or fix branch

Make focused, minimal changes

Include a clear description of:

What was changed

Why it was changed

What risk it addresses or improvement it provides

Submit a pull request for review.

Large changes should be discussed before implementation.

7. Review & Acceptance

All contributions are reviewed by:

AICV maintainers

Or the founder directly

We evaluate:

Alignment with product philosophy

Risk impact

Simplicity

Long-term maintainability

We reserve the right to:

Reject any contribution

Modify contributions

Accept partial contributions

Without obligation to merge.

8. Intellectual Property

By submitting a contribution, you agree that:

Your contribution becomes the exclusive intellectual property of AICV

You grant AICV a perpetual, irrevocable right to use, modify, and distribute your contribution

You retain no licensing or ownership claims over accepted contributions

This is required to maintain coherent product ownership.

9. Confidentiality

You may not:

Share unreleased features

Share internal architecture

Share business strategy

Represent yourself as an owner or decision-maker

Unless explicitly authorized by AICV.

10. Code of Conduct

All contributors are expected to:

Be professional

Be respectful

Avoid harassment or hostility

Focus on the work, not personal disputes

We reserve the right to refuse or terminate collaboration at any time.
